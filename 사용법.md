## 3. 가상환경 및 패키지 설치

### 3-1. Poetry로 의존성 설치

```bash
# pyproject.toml 기반 가상환경 생성 및 의존성 설치
poetry install
```

`poetry install` 이 완료되면 `.venv/` 폴더 또는 Poetry 캐시 경로에 가상환경이 생성됩니다.

가상환경 경로 확인:
```bash
poetry env info --path
# 예시: /Users/yourname/Library/Caches/pypoetry/virtualenvs/sentiment-chatbot-xxxx
```

### 3-2. PyTorch 별도 설치

PyTorch는 CUDA 버전에 따라 설치 명령이 다르므로 별도로 설치합니다.

```bash
# CPU 전용 (Mac / CUDA 미지원 환경)
poetry run pip install torch torchvision torchaudio

# CUDA 11.8 (NVIDIA GPU 환경)
poetry run pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cu118

# CUDA 12.1
poetry run pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cu121
```

> **추가 의존성**: AI Hub에서 Excel 파일을 직접 읽으려면 `openpyxl`이 필요함.
> `poetry add openpyxl` 을 실행해 주세요. 이 패키지는 `scripts/preprocess.py`
> 가 `.xlsx` 파일을 처리할 때 사용됩니다.

설치 확인:
```bash
poetry run python -c "import torch; print(torch.__version__, torch.cuda.is_available())"
# 예시 출력: 2.3.0+cpu False  (CPU 환경)
# 예시 출력: 2.3.0+cu118 True  (GPU 환경)
```

## 5. 데이터 준비

감정 분류 모델 학습을 위한 데이터를 준비합니다.  
아래 세 가지 방법 중 하나를 선택합니다.

### 방법 A — 샘플 데이터 (테스트용, 20개, 즉시 실행 가능)

```bash
poetry run python scripts/preprocess.py --source sample
```

출력 파일:
```
data/processed/train.csv   (16개)
data/processed/val.csv     (2개)
data/processed/test.csv    (2개)
```

### 방법 B — NSMC (네이버 영화 리뷰, 150,000개, 권장)

인터넷 연결이 필요하며 HuggingFace datasets에서 자동으로 다운로드됩니다.

```bash
poetry run python scripts/preprocess.py --source nsmc
```

> `nsmc` 데이터는 긍정/부정 2-class이므로 자동으로 중립(label=1) 없이  
> 부정→0, 긍정→2 로 매핑됩니다.

### 방법 C — 로컬 CSV 파일

자체 수집 데이터를 사용할 경우:

```bash
poetry run python scripts/preprocess.py \
    --source local \
    --data_path /path/to/your/data.csv \
    --text_col text \
    --label_col rating
```

### 추가 자료 — AI Hub 공공데이터

한국어 쇼핑몰 리뷰, 화장품 후기, 고객상담 로그 등 대규모 데이터는
AI Hub 공공데이터포털에서 무료로 내려받을 수 있습니다. 각 항목은 ZIP
형식으로 제공되며, 웹 페이지에서 "파일키 복사" 버튼을 눌러
파일키(fileKey)를 얻을 수 있습니다. 예를 들어:

```bash
curl -L -o fashion.zip \
     "https://www.data.go.kr/download/jsp/download.jsp?fileKey=502205" \
     -H "Authorization: Bearer $AIHUB_API_KEY"
```

`$AIHUB_API_KEY`는 AI Hub 사이트에서 발급받은 인증 토큰입니다. 파일키를 얻은 후 실제로 데이터와 연동하려면 다음 절차를 따릅니다:

1. `curl` 혹은 `wget` 명령에 `fileKey`와 인증 토큰을 전달하여 ZIP을 다운로드
   ```bash
   export AIHUB_API_KEY="your-token-here"
   curl -L -o reviews.zip \
        "https://www.data.go.kr/download/jsp/download.jsp?fileKey=502205" \
        -H "Authorization: Bearer $AIHUB_API_KEY"
   ```
2. 받은 ZIP을 풀어서 나온 CSV/엑셀 파일을 `data/raw/`로 옮깁니다.
   압축 내부에 여러 개가 있거나 `.xlsx` 형식인 경우도 있습니다:
   ```bash
   unzip reviews.zip -d data/aihub_dataset/      # 임시로 풀어두기
   # 또는 게시한 구조대로 이미 Training/Validation 폴더에 넣어둘 수 있음
   ```
   ↓
   ```bash
   # 하나의 CSV로 합치거나, xlsx를 CSV로 변환
   python - <<'PY'
   import pandas as pd, glob
   files = glob.glob('data/aihub_dataset/**/*.xls*', recursive=True)
   out = []
   for f in files:
       if f.lower().endswith(('.xls', '.xlsx')):
           out.append(pd.read_excel(f))
       else:
           out.append(pd.read_csv(f))
   pd.concat(out, ignore_index=True).to_csv('data/raw/aihub_reviews.csv', index=False)
   PY
   ```
   ※ `preprocess.py`는 이제 디렉터리, `*.xlsx`, ZIP 파일도 받아서 내부를
   자동으로 읽습니다. 따라서 `--csv_path`에 `data/aihub_dataset` 혹은
   개별 ZIP 경로를 지정해도 됩니다.
3. `preprocess.py`를 실행하여 학습용 CSV로 변환
   ```bash
   # (디렉터리를 지정하면 안의 모든 CSV/XLSX/ZIP을 모아서 처리)
   poetry run python scripts/preprocess.py \
           --source local \
           --csv_path data/aihub_dataset
   ```
   위 명령은 `data/processed/train.csv`, `val.csv`, `test.csv`를
   8 : 1 : 1 비율로 만들어줍니다. 별도로 ‘validation 폴더’를 관리할
   필요가 없습니다.
4. 이후 `scripts/train.py`로 학습을 진행하세요.

> 웹 UI에서 한 번에 파일을 내려받은 경우에도 같은 디렉터리에 옮기고
> `preprocess.py`를 실행하면 동일하게 연동됩니다.

(명시적으로 AI Hub API를 이용하는 대신 웹 UI에서 수동으로 내려받아도
무방합니다.)

CSV 파일 형식 예시:

| text | rating |
|------|--------|
| 배송이 너무 느려요 | 0 |
| 그냥 평범해요 | 1 |
| 정말 만족스럽습니다 | 2 |

또는 한글 레이블 사용 시:

| text | rating |
|------|--------|
| 배송이 너무 느려요 | 부정 |
| 그냥 평범해요 | 중립 |
| 정말 만족스럽습니다 | 긍정 |

> `--label_col` 의 한글 레이블은 자동으로 0/1/2 로 변환됩니다.

## 6. 모델 학습

### 6-1. 기본 학습

```bash
poetry run python scripts/train.py
```

학습 인자는 `.env` 의 `TRAIN_BATCH_SIZE`, `TRAIN_EPOCHS`, `TRAIN_LR` 값을 자동으로 읽습니다.

### 6-2. 커맨드라인에서 하이퍼파라미터 직접 지정

```bash
poetry run python scripts/train.py \
    --epochs 10 \
    --batch_size 32 \
    --lr 3e-5 \
    --max_length 128
```

| 인자 | 기본값 | 설명 |
|------|--------|------|
| `--epochs` | 5 | 총 에폭 수 |
| `--batch_size` | 16 | 배치 크기 |
| `--lr` | 2e-5 | 학습률 |
| `--max_length` | 128 | 토크나이저 최대 길이 |
| `--warmup_ratio` | 0.1 | warmup 비율 |
| `--seed` | 42 | 랜덤 시드 |

### 6-3. 학습 중 출력 예시

```
Epoch 1/5 | Loss: 0.8821 | Val F1: 0.6234 | [새 최고 모델 저장]
Epoch 2/5 | Loss: 0.5943 | Val F1: 0.7891 | [새 최고 모델 저장]
Epoch 3/5 | Loss: 0.4327 | Val F1: 0.8512 | [새 최고 모델 저장]
Epoch 4/5 | Loss: 0.3801 | Val F1: 0.8601 | [새 최고 모델 저장]
Epoch 5/5 | Loss: 0.3512 | Val F1: 0.8589
학습 완료!  best model → saved_models/sentiment_best.pt
```

### 6-4. 저장되는 파일

```
saved_models/
├── sentiment_best.pt    ← 검증 F1 기준 최고 성능 체크포인트
└── sentiment_last.pt    ← 마지막 에폭 체크포인트
```

> **GPU 메모리 부족 시**: `--batch_size 8` 또는 `--batch_size 4` 로 낮추세요.  
> **학습이 너무 오래 걸릴 때**: `--epochs 3`, NSMC 대신 `--source sample` 로 빠르게 테스트.

## 7. RAG 벡터스토어 구축

`knowledge_base/` 폴더의 `.txt` 파일들을 FAISS 벡터스토어로 변환합니다.

```bash
poetry run python scripts/build_vectorstore.py
```

> `OPENAI_API_KEY` 가 설정되어 있어야 합니다.  
> OpenAI `text-embedding-3-small` 모델을 사용하므로 소량의 API 비용이 발생합니다  
> (faq.txt 기준 약 0.01$ 미만).

### 7-1. 커스텀 문서 추가

`knowledge_base/` 폴더에 `.txt` 파일을 추가하면 자동으로 인덱싱됩니다.

```bash
# 예: 회사 정책 문서 추가
cp /path/to/company_policy.txt knowledge_base/

# 벡터스토어 재빌드
poetry run python scripts/build_vectorstore.py
```

### 7-2. 청크 설정 변경

```bash
poetry run python scripts/build_vectorstore.py \
    --chunk_size 600 \
    --chunk_overlap 100 \
    --top_k 5
```

### 7-3. 생성되는 파일

```
data/vectorstore/
├── index.faiss    ← 벡터 유사도 인덱스
└── index.pkl      ← 문서 메타데이터
```

## 8. FastAPI 서버 실행

```bash
poetry run uvicorn main:app --reload
```

서버 시작 후 아래 URL에서 자동 생성된 API 문서를 확인합니다:

- **Swagger UI**: http://localhost:8000/docs
- **ReDoc**: http://localhost:8000/redoc
- **헬스 체크**: http://localhost:8000/health

### 8-1. 포트 변경

`.env` 에서 `FASTAPI_PORT=9000` 으로 변경하거나:

```bash
poetry run uvicorn main:app --reload --port 9000
```

### 8-2. 프로덕션 실행 (reload 없이)

```bash
poetry run uvicorn main:app --host 0.0.0.0 --port 8000 --workers 2
```

## 10. API 사용 예시

### 단일 감정 분석

```bash
curl -X POST http://localhost:8000/api/v1/analysis/ \
     -H "Content-Type: application/json" \
     -d '{"text": "배송이 너무 늦어요. 정말 실망입니다."}'
```

응답:
```json
{
  "label": 0,
  "label_str": "NEGATIVE",
  "negative": 0.87,
  "neutral": 0.09,
  "positive": 0.04,
  "escalate": true
}
```

### 배치 감정 분석

```bash
curl -X POST http://localhost:8000/api/v1/analysis/batch \
     -H "Content-Type: application/json" \
     -d '{"texts": ["너무 좋아요!", "그냥 평범해요", "다시는 안 살 것 같아요"]}'
```

응답:
```json
[
  {"label": 2, "label_str": "POSITIVE", "negative": 0.02, ...},
  {"label": 1, "label_str": "NEUTRAL",  "negative": 0.08, ...},
  {"label": 0, "label_str": "NEGATIVE", "negative": 0.91, "escalate": true, ...}
]
```

### 챗봇 대화

```bash
curl -X POST http://localhost:8000/api/v1/chat/ \
     -H "Content-Type: application/json" \
     -d '{"session_id": "user-001", "message": "환불 절차가 어떻게 되나요?"}'
```

응답:
```json
{
  "session_id": "user-001",
  "answer": "환불은 반품 수거 확인 후 영업일 기준 3~5일 이내에 처리됩니다. ...",
  "sentiment": {
    "label": 1,
    "label_str": "NEUTRAL",
    "negative": 0.05,
    "neutral": 0.88,
    "positive": 0.07,
    "escalate": false
  },
  "sources": ["반품 수거 확인 후 영업일 기준 3~5일 이내에..."],
  "escalate": false
}
```

### 세션 히스토리 초기화

```bash
curl -X DELETE http://localhost:8000/api/v1/chat/user-001
```

## 13. 자주 발생하는 오류 해결

### `FileNotFoundError: saved_models/sentiment_best.pt`

모델이 아직 학습되지 않은 상태입니다.

```bash
poetry run python scripts/preprocess.py --source sample
poetry run python scripts/train.py --epochs 5
```

---

### `openai.AuthenticationError: Incorrect API key`

`.env` 파일에 올바른 API 키가 입력되지 않았습니다.

```bash
# .env 파일 확인
cat .env | grep OPENAI_API_KEY
```

---

### `ModuleNotFoundError: No module named 'torch'`

PyTorch는 별도 설치가 필요합니다.

```bash
poetry run pip install torch torchvision torchaudio
```

---

### `CUDA out of memory`

GPU 메모리가 부족합니다. 배치 크기를 줄이세요.

```bash
poetry run python scripts/train.py --batch_size 8
```

---

### `faiss.swigfaiss.IOError: Index not found`

벡터스토어가 아직 빌드되지 않았습니다.

```bash
poetry run python scripts/build_vectorstore.py
```

---

### `poetry: command not found`

Poetry가 설치되지 않았거나 PATH에 없습니다.

```bash
export PATH="$HOME/.local/bin:$PATH"
# 또는 재설치
curl -sSL https://install.python-poetry.org | python3 -
```

## 14. HuggingFace Spaces 배포

Gradio 데모를 HuggingFace Spaces에 무료로 배포할 수 있습니다.

### 14-1. Spaces 생성

1. [huggingface.co/new-space](https://huggingface.co/new-space) 접속
2. Space name: `sentiment-chatbot` (원하는 이름)
3. SDK: **Gradio** 선택
4. Visibility: Public (무료) 또는 Private

### 14-2. 필요 파일 추가

```bash
# requirements.txt 생성 (Spaces는 poetry 대신 pip 사용)
cat > requirements.txt << 'EOF'
transformers==4.40.0
datasets==2.19.0
langchain==0.1.20
langchain-openai==0.1.6
faiss-cpu==1.8.0
fastapi==0.115.0
gradio==4.36.0
loguru==0.7.2
pandas==2.2.0
pydantic-settings==2.2.1
torch
EOF
```

### 14-3. 환경 변수 설정

HuggingFace Spaces Settings > Secrets 메뉴에서:

```
OPENAI_API_KEY = sk-xxxxxxxxxxxx
```

### 14-4. app.py 진입점

Spaces는 `app.py` 파일의 `demo` 변수를 자동으로 찾아 실행합니다.  
현재 `app.py` 의 마지막 줄은 Spaces 환경에 맞게 수정합니다:

```python
# Spaces 환경에서는 launch() 호출 없이 demo 객체만 노출
if __name__ == "__main__":
    demo.launch()
else:
    # HuggingFace Spaces 자동 실행
    demo.queue()
```

### 14-5. 배포

```bash
# HuggingFace 저장소와 연결
git remote add space https://huggingface.co/spaces/<USERNAME>/sentiment-chatbot
git push space main
```

빌드 및 배포는 약 3~5분 소요되며, Spaces 페이지에서 로그를 확인할 수 있습니다.
